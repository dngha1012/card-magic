# 카메라 카드 변환 마술 앱 (Card Morph Magic PWA)

## 프로젝트 개요

전면 카메라를 통해 마술사가 들고 있는 실제 카드 위에 다른 카드의 "필터"가 씌워져 보이게 하고, 마술사가 카드 위를 손가락으로 쓸어내리면 필터가 벗겨지며 진짜 카드가 드러나는 라이브 마술 도구.

**플랫폼**: 모바일 웹앱 (PWA) — iOS Safari에서 동작
**사용자**: 개발자 본인 (매슐, TikTok 마술 채널 @abcdefghigklnm0pqrstuvwx)
**용도**: 라이브 마술 공연 및 TikTok 콘텐츠 촬영
**배포**: 앱스토어 미배포, 본인 폰에만 설치 (홈 화면 추가)

## 마술 루틴 (관객 시점)

1. 관객이 덱에서 **자유롭게 카드 한 장 뽑음** (예: 5하트)
2. 마술사가 카드를 덱에 다시 넣고 **컨트롤로 톱(맨 위)에 위치**시킴
3. 카드를 들기 직전 앱 실행 → **검은 화면** (인식 진행 중, 관객은 로딩으로 인식)
4. 카드를 카메라 앞에 들어 올림 → 화면 페이드 인 → **다른 카드(예: 8클로버)로 보임**
5. 마술사가 카드 위를 **검지+중지 두 손가락으로 위→아래 쓸어내림**
6. 손가락이 지나간 자리부터 **필터가 벗겨지며 진짜 카드(5하트)가 드러남**
7. 변환 완료 후 2~3초 카메라에 머무름 → 관객에게 실물 카드 건네줌
8. 관객이 직접 확인 = 진짜 5하트

**왜 강한 루틴인가**:
- 자유 선택 (포스 아님) → 의심 차단
- 화면 변화 + 실물 확인 = 검증 가능한 엔딩
- 세 번의 충격: 시각적 변화 / 인지적 충격(내 카드잖아) / 물리적 확인

## 기술 스택

| 영역 | 기술 |
|------|------|
| 프론트엔드 | HTML / JavaScript (또는 React) |
| 카메라 | MediaDevices API (getUserMedia) |
| 카드 영역 검출 | OpenCV.js (직사각형 추적, 호모그래피) |
| 카드 종류 인식 | TensorFlow.js + Roboflow 공개 모델 |
| 손 추적 | MediaPipe Hands (TF.js 버전) |
| 렌더링 | Canvas / WebGL |
| 배포 | 정적 호스팅 (GitHub Pages 등) → Safari에서 "홈 화면 추가" |

## 핵심 설계 결정

### 카드 인식 전략
- **A-1 경로**: Roboflow Universe의 공개 플레잉 카드 모델 그대로 사용
- 바이시클 표준 카드 사용 → 데이터셋 호환성 높음
- 추가 학습은 A-1이 부족할 때만 (촬영 환경 특화)

### 인식 타이밍 (검은 화면 트릭)
- 앱 실행 시 검은 화면 + 로딩 인디케이터 (예: "focusing...")
- 백그라운드에서 카메라 작동 + 카드 인식
- **여러 프레임(10~20) 연속으로 같은 카드** 감지되면 인식 확정
- 확정 시 랜덤으로 필터 카드 결정 (뽑힌 카드 제외 51장 중 1장)
- 페이드 인 → 이미 필터 입혀진 상태로 등장
- 인식 후 매 프레임 재분류 X, 위치 추적만 (성능/안정성)

### 손가락 스와이프 인식
- MediaPipe Hands로 두 손 동시 추적
- 카드 잡은 손 / 쓰는 손 자동 구분
- 쓰는 손의 **검지(landmark 8)와 중지(landmark 12) 끝** 좌표 사용
- 두 손가락 끝을 잇는 선의 Y좌표가 위→아래로 이동할 때 필터 점진적 마스킹
- 손가락 지나간 영역은 영구적으로 벗겨진 상태 유지

### 자연스러움 디테일
- 필터 카드 가장자리는 실제 카드 안쪽으로 살짝 들어가게 (손가락 가림 방지)
- 필터 벗겨지는 경계는 일자 X, **약간 흐트러진 가장자리** (안개/파티클 느낌)
- 변환 직후 2~3초 머무름 (관객의 "화면 속 그 카드 = 실물 그 카드" 인식 확보)

## 동작 흐름 다이어그램

```
[앱 실행]
    ↓
[검은 화면 + 로딩]  ← 카메라 ON, 인식 시작
    ↓
[카드 인식 확정]    ← 10~20 프레임 일관성 체크
    ↓
[랜덤 필터 카드 결정]  ← 뽑힌 카드 제외 51장 중 1
    ↓
[페이드 인]
    ↓
[필터 적용 상태]    ← 호모그래피 와핑으로 카드 면에 가짜 카드 얹기
    ↓
[손가락 진입 감지]  ← MediaPipe로 카드 위 손 인식
    ↓
[스와이프 추적]     ← 검지+중지 Y좌표 위→아래 이동
    ↓
[점진적 마스킹]     ← 손가락 지나간 자리 필터 제거
    ↓
[변환 완료 + 2~3초 머무름]
    ↓
[실물 카드 관객에게]
```

## 개발 로드맵 (단계별)

각 단계는 **독립적으로 작동 확인 후 다음으로**. 한 번에 다 만들지 않음.

### 1단계: 카메라 켜기
- HTML/JS로 전면 카메라 영상을 화면에 띄우는 페이지
- iOS Safari에서 카메라 권한 + 영상 표시 확인

### 2단계: 카드 영역 검출 (필터 없이)
- OpenCV.js로 직사각형 검출
- 카드 위에 빨간 박스가 따라다니면 성공

### 3단계: 카드 종류 인식
- Roboflow 공개 모델 + TensorFlow.js
- 5하트 들면 화면에 "5H" 텍스트 표시

### 4단계: 필터 오버레이 (호모그래피 와핑)
- 검출된 카드의 네 모서리에 다른 카드 이미지를 와핑해서 덮기
- 5하트 들었는데 화면엔 8클로버 보이면 성공
- **가장 까다로운 단계 — 시간 많이 걸릴 것**

### 5단계: 검은 화면 → 페이드 인 연출
- 시작 시 검은 화면 + 로딩
- 인식 확정 후 페이드 인
- 인식 결과 고정 (이후 위치 추적만)

### 6단계: MediaPipe로 손가락 추적
- 두 손 동시 추적
- 카드 잡은 손 / 쓰는 손 구분
- 검지+중지 끝 좌표 실시간 표시

### 7단계: 손가락 경로 따라 필터 마스킹
- 손가락 Y좌표 위→아래 이동에 맞춰 필터 점진 제거
- 지나간 영역 영구 마스킹 유지

### 8단계: 디테일 다듬기
- 벗겨지는 경계 파티클/안개 효과 (WebGL 셰이더)
- 가짜 카드에 광택/그림자 입히기
- 페이드 / 머무름 타이밍 조정
- 실패 시 백업 시나리오 (타임아웃 후 그냥 진짜 카드 표시)

## 가장 어려울 부분 (미리 알기)

1. **4단계 호모그래피 와핑** — 카드가 휘어지거나 기울어질 때 필터 카드도 똑같이 변형. OpenCV.js의 `findHomography` + `warpPerspective` 활용.
2. **7단계 스와이프 마스킹의 자연스러움** — 디지털 티 안 나게 가장자리 처리.
3. **iOS Safari 호환성** — 일부 카메라 API가 제약적. 매번 실기기 테스트 필수.

## 백업 / 폴백 전략

- 인식 타임아웃(예: 2초) 시 → 필터 없이 진짜 카드 그대로 표시 (마술 실패하지만 앱은 안 망가짐)
- 손가락 추적 실패 시 → 시간 경과 후 필터 자동 페이드 아웃
- 촬영 환경(조명, 배경)을 일정하게 유지해서 인식 환경 통제
